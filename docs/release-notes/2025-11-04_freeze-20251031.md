# Release Notes – Freeze 20251031 (04/11/2025)

Versione applicazione: 1.0.1-dev
Branch: freeze-20251031

## Novità e modifiche

- UI test popup
  - Rimosso il riquadro dettagli "Test 0: Pre check".
  - Aggiunto tag compatto di stato nel header (in corso / OK / KO) per liberare spazio ai risultati E2E.
  - Mantiene semantica "strict": i risultati E2E compaiono solo a fine esecuzione Playwright.

- Testing e qualità
  - Aggiunta configurazione Vitest (`vitest.config.ts`) per escludere i test E2E Playwright dalla suite Vitest.
  - Endpoint API dei test esteso con suite "smoke": `POST /api/run-tests?suite=smoke` esegue solo il controllo rapido su `/api/version`.
  - Runner E2E aggiornato (`src/tests/runE2E.js`) per gestire `suite` (full/smoke) e invocare Playwright con filtro adeguato.

- Operatività server
  - Aggiunto endpoint di spegnimento "graceful": `POST /api/shutdown`.
    - Chiude il server ed esce con codice 0, evitando exit code -1 nelle pipeline locali.

## Performance

- Smoke E2E (suite=smoke):
  - Tempi osservati: ~9.3–10.0 secondi su ambiente locale.
  - Note: l'overhead è principalmente dovuto all'avvio del runner Playwright e del browser di test.

## Quality gates

- Build: PASS (nessuna build necessaria per queste modifiche)
- Test (Vitest): PASS (test unitari/di base)
- E2E Smoke via API: PASS (~9–10s)
- Lint/Typecheck: N/A

## Compatibilità e note di upgrade

- Nessuna migrazione DB richiesta.
- Le API aggiunte sono retrocompatibili:
  - `/api/run-tests` accetta facoltativamente `?suite=smoke` oltre al comportamento predefinito (full).
  - `/api/shutdown` è opzionale e utile per automazioni/script locali.
